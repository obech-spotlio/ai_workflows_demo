# SS-3614: Project Scaffolding & Configuration

## Ticket Summary

Initialize a Next.js 15 project with App Router, TypeScript strict mode, Tailwind CSS 4, ESLint 9, Prettier, and Vitest. Configure all tooling, directory structure, and npm scripts. Data will be persisted via localStorage with no database.

**Status:** To Do
**Type:** Story
**Project:** (Dev) DevOps (SS)

## Acceptance Criteria Checklist

- [ ] Next.js 15 app bootstrapped with App Router
- [ ] TypeScript strict mode enabled with `noUncheckedIndexedAccess` and `exactOptionalPropertyTypes`
- [ ] Tailwind CSS 4 configured with Stitch design system preset
- [ ] ESLint 9 + Prettier configured
- [ ] Vitest + Testing Library configured
- [ ] Package.json scripts: `dev`, `build`, `lint`, `format`, `test`, `test:watch`
- [ ] tsconfig paths alias `@/*` → `./src/*`
- [ ] Feature-sliced directory structure created:
  - `src/app`

  - `src/shared`
  - `src/shared/ui`

## Implementation Approach

1. **Next.js 15 Bootstrap**
   - Use `create-next-app@15` with App Router template, or manually set up the latest Next.js 15
   - Configure `next.config.ts` if needed for custom builds

2. **TypeScript Configuration**
   - Set `strict: true` in `tsconfig.json`
   - Enable `noUncheckedIndexedAccess: true` and `exactOptionalPropertyTypes: true`
   - Add path alias: `"@/*": ["./src/*"]`

3. **Tailwind CSS 4 Setup**
   - Install Tailwind CSS 4 (compatible with Next.js 15)
   - Integrate Stitch design system preset
   - Configure `tailwind.config.ts`
   - Ensure PostCSS is properly configured

4. **ESLint 9 & Prettier**
   - Install and configure ESLint 9 (with Next.js plugin if needed)
   - Set up Prettier with ESLint integration
   - Create `.eslintrc.json` and `.prettierrc` configs
   - Add `lint` and `format` scripts to package.json

5. **Vitest + Testing Library**
   - Install Vitest and @testing-library packages
   - Configure `vitest.config.ts` with Next.js/React setup
   - Add `test` and `test:watch` scripts

6. **Directory Structure**
   - Create feature-sliced architecture directories:
     - `src/app` - Next.js App Router pages
     - `src/features` - feature modules
     - `src/shared` - shared utilities and components
     - `src/shared/ui` - reusable UI components

7. **Package.json Scripts**
   - `dev` - Start development server
   - `build` - Build for production
   - `lint` - Run ESLint
   - `format` - Format code with Prettier
   - `test` - Run Vitest once
   - `test:watch` - Run Vitest in watch mode
